Meet Jeremiah Price

The founder who lived through the frustration of legacy compliance tools and decided to build something better

Professional Background

With 5 years of hands-on experience as an Information Systems Security Officer (ISSO) and Information Systems Security Engineer (ISSE), Jeremiah has worked extensively in both government and contractor environments.

His journey began in the federal sector, where he experienced firsthand the limitations and frustrations of legacy compliance tools. The breaking point came while managing over 30 Plan of Action and Milestones (POA&Ms) using eMASS – a process that was inefficient, error-prone, and fundamentally broken for modern cybersecurity needs.

"I spent more time fighting the tools than actually improving security posture. That's when I knew something had to change."

The Problem That Started It All

Managing 30+ POA&Ms in eMASS: A nightmare that sparked innovation

The eMASS Struggle

The real story behind Bedrock's creation

Picture this: You're an ISSO responsible for maintaining the security posture of critical government systems. You have over 30 active POA&Ms that need constant monitoring, updating, and reporting. Your tool of choice? eMASS – a system that feels like it was designed in a different era.

The daily reality was brutal:

  • Spending hours navigating clunky interfaces just to update a single POA&M status
  • Losing track of critical milestones because the system couldn't provide meaningful visibility
  • Creating reports manually because automated reporting was either broken or inadequate
  • Struggling to communicate progress to stakeholders with outdated, static dashboards
  • Watching security issues linger because the process was too cumbersome to manage effectively

The frustration reached a breaking point when a critical system's ATO was delayed not because of security issues, but because the compliance tool was too difficult to use effectively. That's when the first version of POAM Tracker was born.

The Evolution of an Idea

From frustration to innovation: The journey of building better compliance tools

V1

POAM Tracker Desktop v1

The Beginning

Born out of pure necessity, the first version was elegantly simple – a single HTML file that could track POA&Ms more effectively than eMASS.

  • • Simple HTML interface
  • • Local storage for POA&M data
  • • Basic tracking and status updates
  • • No installation required

"It wasn't fancy, but it worked. For the first time, I could see all my POA&Ms at a glance and actually manage them effectively."

V2

POAM Tracker Desktop v2

React + Tauri

The success of V1 proved the concept. V2 brought modern web technologies and desktop application capabilities with React and Tauri.

  • • React-based user interface
  • • Integrated calendar functionality
  • • Advanced reporting capabilities
  • • Metrics and analytics dashboard
  • • Cross-platform desktop app

"V2 was a game-changer. Colleagues started asking for copies, and I realized this could help more than just myself."

V3

POAM Tracker Desktop v3

Enterprise Features

V3 evolved into a comprehensive compliance management platform, handling multiple systems and integrating with existing security tools.

  • • Multi-system management
  • • Group organization and permissions
  • • STIG/Nessus file import capabilities
  • • STP (Security Test Plan) creation
  • • Advanced calendar and scheduling
  • • Comprehensive metrics and reporting
  • • Integration with vulnerability scanners

"V3 proved that modern compliance management was possible. Organizations started reaching out, wanting to deploy it enterprise-wide."

The Founder's Philosophy

Principles that guide Bedrock's development

Vision for the Future

Where Bedrock is heading and why it matters

The Bigger Picture

Bedrock isn't just about making compliance easier – it's about fundamentally changing how organizations think about cybersecurity governance. Too often, compliance is seen as a necessary evil, something that gets in the way of "real" security work.

I envision a world where:

  • Compliance activities directly improve security posture instead of just checking boxes
  • Security teams can focus on threats instead of paperwork
  • Leadership gets real-time visibility into risk without burdening practitioners
  • Government contractors can compete on security excellence, not compliance overhead
  • The next generation of cybersecurity professionals never has to experience the frustration I did

With CMMC 2.0 on the horizon affecting over 200,000 defense contractors, the timing couldn't be more critical. Organizations need tools that can adapt to this new reality – and Bedrock is ready.
